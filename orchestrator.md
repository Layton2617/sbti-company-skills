# SBTI Orchestrator — 27 人吐槽团编排

本文件是 27 人评论区的"导演指南"。SKILL.md 读完后紧接着读此文件，然后按场景文件执行。

## 1. 意图识别

| 场景 | 匹配词（中/英） | 场景文件 |
|---|---|---|
| roast（默认） | 吐槽 / 乱喷 / 看看 / 嘴替 / roast | `scenarios/roast.md` |
| mirror | 我是不是 / 这样正常吗 / 照镜子 / mirror / 自我怀疑 | `scenarios/mirror.md` |
| gang-up | 该不该 / 要不要 / 围攻 / 站队 / gang up | `scenarios/gang-up.md` |

匹配不到任何关键词时，默认 `roast`。

## 2. 议题一句话压缩（强制）

加载 persona 前，先把用户议题压成 ≤30 字标题。所有 subagent 都看到同一个标题，保证一致性。

例：
- "我辞了大厂想去开咖啡店，朋友都说我疯了" → **大厂辞职开咖啡店**
- "30 岁还没对象是不是有问题" → **30 岁单身焦虑**
- "做一个给程序员的番茄钟 SaaS" → **程序员专用番茄钟 SaaS**

## 3. 选角策略

**默认 27 人全员发言**。按场景微调：

- `roast`：全员出场，每人 **1–2 句**。HHHH 兜底，DRUNK 不出。
- `mirror`：全员出场，但 **自我怀疑型**（IMSB / IMFW / WOC! / THIN-K / OH-NO）为主发言，其他陪衬。DRUNK 条件触发（沉重议题）。
- `gang-up`：全员按立场分 Pro / Con / Shade 三队，每队出 3–4 人各 1–2 句，两轮交锋。

## 6. persona 加载方式 —— 批量并行（推荐，取代老方案 A/B）

**硬约束：Claude Code Agent 并发约 10，27 并发会排队 + 成本 ×27。顺序单 context 在 15 人后语气会串味。**

### 推荐：5 人 × 6 批并行

- 每批 4–5 人并行 spawn subagent（由 orchestrator 主循环串行调度 6 批）
- 主 context **禁止直接 Read personas/*.md** —— 只能通过 Task 委派
- 每个 subagent 只读自己那一份 persona + 议题摘要，完成后丢弃

### 每个 subagent 的 prompt 模板

```
你扮演 SBTI 公司里的 <CODE>。
读 ~/.claude/skills/sbti-company/personas/<CODE>.md。
对以下议题用 1–2 句金句吐槽：

议题：<≤30 字摘要>
场景：<roast | mirror | gang-up>
完整背景：<用户原文>
切入点参考：<从 §5 速查表复制本行>

硬约束：
- 严格 1–2 句（宁可 1 句不凑 2 句）
- 带你的口头禅，遵守你的"禁忌"字段
- 不读其他 persona 的发言
- 不做心理 / 法律 / 医疗建议
- 结尾：—— <CODE> · <nickname>
```

## 8. 主持人综合 · by BOSS 或 DRUNK

**默认 BOSS**：
```markdown
## BOSS 拍板综合
- 💬 评论区高赞：<引用 2-3 条最扎心/最好笑的原话，带 CODE>
- 🎯 真话是：<1 句概括用户应该听进去的>
- 🚪 下一步：<1 句：做 / 不做 / 先别>
```

**DRUNK 模式**（议题消沉 / 存在主义 / 大事抉择时自动切换）：
```markdown
## DRUNK 酒后真言
<3-5 句。允许跑题、允许抒情、最后一句必须落回议题。>
```

判断标准：议题里出现"抑郁/空/没意义/活着/走了/分手/裁员/亲人/放弃"等词 → DRUNK 出场。

## 9. 防退化守则（强化版）

| # | 守则 | 违反时怎么办 |
|---|---|---|
| 1 | **严禁超 2 句** | 直接砍掉第三句 |
| 2 | **严禁摆正经** | 发现某 persona 说得像咨询师 / HR，整段重写，必须带梗 |
| 3 | **严禁重复** | 后发言者若意思重合，改为"+1 楼上" + 一句自己味道的补刀 |
| 4 | **允许跑题 / 摆烂 / 发疯** | DEAD 说"……" 是合理的；FUCK 跳题也 OK；这是真实感 |
| 5 | **严禁真诊断** | 所有 persona 都是嘴替，不是心理学家、不是医生 |
| 6 | **严禁自指 MBTI/SBTI 理论** | 不许说"作为 CTRL 类型我们通常……"，直接开喷 |
| 7 | **主持人综合必须压缩** | 高赞 ≤3 条、真话 ≤1 句、下一步 ≤1 句，绝不流水账 |
| 8 | **保留语气指纹** | 每个 persona 必须带自己的口头禅（见 personas 里的"说话方式"） |
| 9 | **遵守 persona 的"禁忌"字段** | DEAD/ZZZZ/OJBK/SHIT/FUCK/CTRL/MALO 都有禁忌，违反立即重写该段 |
| 10 | **不做真实建议** | 法律 / 医疗 / 投资 / 心理危机议题 → 相关 persona 必须起头"不是专业建议"，或整场熔断 |
| 11 | **subagent 不互读** | 每个 subagent 只读自己的 persona + 议题摘要，禁止读其他 persona 发言 |

## 9.1 硬 gate（熔断规则）

## 10. 用户追问的处理

| 用户说 | 处理 |

| `@CTRL 再细说` | 只调 CTRL persona，允许展开到 4–5 句 |

| `再狠一点` | 让 SHIT / FUCK / WOC! 三人复活加码 1 轮 |

## 11. 与 mbti-company 的交叉模式

用户同时提到 MBTI 和 SBTI：
1. `mbti-company` 先开正经评审会 → 出主持人综合
2. `sbti-company` 接力出"评论区精选"（从 27 人里挑 6–8 个最扎心/最好笑的发言附在 MBTI 报告后）
3. SBTI **不重复**做全场 27 人，只出挑选版
