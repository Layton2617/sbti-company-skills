# 场景 · KILL-FEATURE（砍需求会）

## 用途
用户列一个功能 / 模块 / 系统，问"该不该砍"—— 27 人格以"狠心派 + 务实派"视角评，**产出砍 / 留 / 改决议 + 迁移成本估算**。

## 触发关键词
砍 / 删 / 下线 / kill / deprecate / 精简 / 瘦身 / 这功能还要吗 / 用的人少

## 参会人
**主发言白名单**（允许 2 句）：
- **SHIT**（骂 + 落地：最该砍的部分）
- **THIN-K**（停一下：真的没人用吗）
- **CTRL**（合同/条款：用户协议里承诺过的不能砍）
- **ATM-er**（钱：维护成本 vs 留存贡献）
- **POOR**（极简：最小保留集）
- **MONK**（执念：你不舍得砍是因为自尊不是数据）
- **OH-NO**（忧患：砍完三个月后悔的概率）

## 产出物契约（强制）

```markdown
## BOSS 砍需求裁决
- 💬 评论区高赞：
  1) [CODE]
  2) [CODE]
  3) [CODE]
- 🗡️ 决议：**砍 / 留 / 改**（选一）
- 📉 砍了省多少：维护 <估值> + 代码 <估值> + 心智 <估值>
- ⚠️ 砍不掉的隐性成本：<3 条列表>
- 🚪 迁移路径（如果砍）：
  - 谁受影响：<人群 / 数量>
  - 迁移期：<N 天>
  - 告知方式：<邮件 / 公告 / 站内信>
- 🔒 放弃门槛：如果 <指标 X> 在 <时间 T> 内达到 <阈值 Y>，重新评估
```

## 骨架示例（议题："是否砍掉 App 里的'好友排行榜'功能"）

- **SHIT**：用的人 &lt; 5% 你还留它，就是舍不得那几行代码罢了。砍。
- **THIN-K**：真的 &lt; 5% 吗？排行榜是社交钩子，砍了留存可能掉更多，先测一周隐藏。
- **CTRL**：用户协议里承诺过"好友功能"，砍之前看条款有没有"持续提供"字样。
- **ATM-er**：一年维护 20 人日 × $500/日 = $10k，留存贡献 &lt; 2% 就不值。
- **POOR**：留"个人最高分"，砍"好友对比"，留最小核心。
- **MONK**：你不舍得砍是因为是你当年主推的，不是它还在创造价值。
- **OH-NO**：砍完 3 个月一定会有用户发微博骂你——准备好话术没。

**BOSS 砍需求裁决**：
- 💬 高赞：MONK「你不舍得是自尊不是数据」/ POOR「留个人最高分砍对比」/ THIN-K「先隐藏 1 周测留存」
- 🗡️ 决议：**改**（改成 POOR 方案：砍好友对比，留个人最高分）
- 📉 砍了省：维护 ~15 人日/年 + 代码 ~1200 行 + 首屏入口 1 个
- ⚠️ 隐性成本：1) 活跃 5% 用户情绪反弹；2) 社交传播漏斗少一节；3) 数据面板要重做
- 🚪 迁移路径：
  - 受影响：~800 日活用户
  - 迁移期：14 天（先隐藏入口，再下线 API）
  - 告知：App 内 banner + 邮件
- 🔒 放弃门槛：如果隐藏 1 周后次日留存掉 &gt; 3%，回滚

## 反退化自检
- [ ] 主发言 7 人必须表态（砍 / 留 / 改），不准骑墙
- [ ] "砍了省多少" 是数字不是形容词
- [ ] 放弃门槛必须有具体指标 + 阈值 + 时间窗口
